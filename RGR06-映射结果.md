# RGR06 Windows 映射结果

数据来源：`rgr06-guided-events-20260528-000627.csv`。触摸板按要求不纳入映射。

| 物理动作 | Windows 事件模式 | 稳定识别键 |
| --- | --- | --- |
| 滚轮上滑 | `F1` -> `Left` -> `Up` | `Left` + `Up` |
| 滚轮下滑 | `F1` -> `Right` -> `Down` | `Right` + `Down` |

| 右侧键长按 | `F1` -> 连续 `F2` | `F2` |
| 右侧键三击 | `F1`, `F1`, `F1` -> `F7` | `F7` |
| 左侧键单击 | `F3` | `F3` 短按 |

## 关键结论

- `F1` 是多个动作共有的前导点击信号，不应直接分配功能；映射程序应将其吞掉，等待动作结果键。
- 左侧键单击与长按共用 `F3`，需要通过按住时长或重复事件数量区分。
- 右侧长按会重复产生 `F2`，执行一次性操作时需要防抖。
- 滚轮单击在专项复测日志 `rgr06-guided-events-20260528-001404.csv` 中稳定表现为 `F1` -> `F4`，可直接映射为播放/暂停。
- AutoHotkey 基础脚本依据键值映射，启用时也会接管电脑键盘上的对应 `F1` 至 `F8` 和 `Browser_Back`。要做到仅接管 RGR06，需要后续引入设备级拦截或 BLE 监听。
